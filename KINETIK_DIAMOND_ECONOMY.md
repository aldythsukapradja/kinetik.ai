# Kinetik Diamond Economy

This is the source of truth for connecting every Kinetik app to the real diamond
economy.

Read this before changing rewards, Buddy progress, app event emissions,
leaderboards, quests, or shop/cosmetic systems.

## 1. Economy Thesis

Diamonds should not be loose points inside each app.

Diamonds are a **circle-wide emotional economy**:

```text
Meaningful app action
  -> KINETIK_EVENT
  -> Economy policy
  -> cap / approval / duplicate check
  -> diamond event
  -> Buddy state
  -> boards / quests / closet / suggestions
```

The app proposes what happened. The shell decides what it is worth.

## 2. Economy Principles

1. Apps do not mint core diamonds directly.
2. Apps emit approved event types through `KINETIK_EVENT`.
3. The shell is the economy authority.
4. Rewards must explain why they happened.
5. Child chore rewards need adult approval.
6. Low-value repeat actions are capped.
7. Specialized apps can have sidecar badges without affecting core diamonds.
8. Moments are mostly Story Energy, not diamond-heavy.
9. Calendar rewards are small because scheduling can be spammed.
10. The same economy must scale from localStorage to Google Sheets to Supabase.

## 3. Core Units

| Unit | Meaning | Source |
|---|---|---|
| Diamonds | Spendable / visible progress currency | `diamond_events` |
| XP | Level progress for Buddy | derived from diamonds and special events |
| Energy | Category flavor of progress | event policy |
| Mood | Buddy emotional state | latest meaningful events |
| Level | Buddy growth stage | total XP |
| Badges | Non-spendable achievements | milestone events |
| Quests | Daily/weekly goals | generated by shell/Buddy |
| Inventory | Cosmetics, rooms, trophies | purchased/unlocked from diamonds or badges |
| Leaderboards | Comparison boards | derived snapshots, not raw wallet authority |

## 4. Energy System

| Energy | Category | Meaning | Examples |
|---|---|---|---|
| Care | household / responsibility | helping the circle | Home Quest, Grocery Run, Kitchen Buddy |
| Growth | learning / coaching / reflection | building skills | Times Quest, Primary Quest, coaches |
| Play | games / fun | light play and friendly competition | Circle Chess, Code Clash, Emoji Party |
| Move | sport / physical practice | body and performance | Padel, Tennis, Basketball, Guitar practice if treated as performance |
| Circle | social / coordination | planning and group rhythm | Calendar, Chat, polls, Padel Matchday booking |
| Story | memories / media | family story and identity | Moments, Cinema |

## 5. Event Contract

Apps should emit only these core event types unless the economy doc is updated.

| Event type | Use | Reward posture |
|---|---|---|
| `task.completed` | child or member says a task is done | pending/no diamonds if approval required |
| `task.approved` | adult/leader approves a task | medium Care diamonds |
| `practice.completed` | lesson, drill, rehearsal, sport rep | small Growth/Move diamonds |
| `mastery.unlocked` | level/crown/objective/mastery milestone | larger capped diamonds |
| `game.round.completed` | casual round completed | tiny Play diamonds |
| `game.win` | competitive win or scored session win | capped Play/Move diamonds |
| `calendar.created` | useful event/routine created | tiny Circle diamonds |
| `reflection.saved` | meaningful note, journal, coaching reflection | tiny Growth diamonds |
| `moment.created` | family memory posted | Story Energy; usually no core diamonds |
| `moment.styled` | album/card/recap/theme created | Story Energy; optional tiny reward |
| `buddy.quest.completed` | Buddy-generated quest finished | quest-specific reward |

Deprecated / migration-only:

| Old event/message | Replace with |
|---|---|
| `EARN_GEMS` | `KINETIK_EVENT` with a specific event type |
| `diamonds.earned` | `task.approved`, `practice.completed`, etc. |
| app-local `gems` as authority | app-local UI only; shell ledger is authority |

## 6. Reward Policy

These values are the first real economy baseline. They can be tuned after usage.

| Event type | Default diamonds | Max per event | Daily cap | Notes |
|---|---:|---:|---:|---|
| `task.completed` | 0 | 0 | 0 | creates pending approval when needed |
| `task.approved` | 8 | 30 | 80 | app may propose chore amount; shell caps it |
| `practice.completed` | 3 | 12 | 45 | 3 to 5 rewarded sessions/day/app/person |
| `mastery.unlocked` | 15 | 40 | 80 | crowns/objectives/boss rounds |
| `game.round.completed` | 2 | 8 | 30 | casual play; no farming |
| `game.win` | 5 | 15 | 35 | competitive but capped |
| `calendar.created` | 2 | 5 | 15 | scheduling is useful but spam-prone |
| `reflection.saved` | 2 | 8 | 25 | requires real note/reflection metadata |
| `moment.created` | 0 | 3 | 10 | Story Energy first; tiny reward optional |
| `moment.styled` | 0 | 5 | 10 | reward creation, not repeated filters |
| `buddy.quest.completed` | 12 | 35 | 80 | depends on quest difficulty |

XP baseline:

```text
buddy_xp_gain = diamonds_awarded * 4
level = floor(total_xp / 120) + 1
```

## 7. Anti-Farming Rules

| Risk | Rule |
|---|---|
| repeating same source record | `sourceRecordId` can only reward once |
| endless easy rounds | cap by app/person/day |
| randomizer farming | random/spinner apps get no major diamonds |
| calendar spam | tiny reward and daily cap |
| child self-approval | `task.completed` waits for adult approval |
| repeated Moment styling | only reward first useful styled output per source |
| sidecar event farming | sidecar apps default to badges/leaderboards, not core diamonds |
| app-local tampering | shell validates event type, amount, cap, person, app |

## 9. Priority Wiring Waves

### Wave 1 - Make the economy real

Wire these first:

1. Kinetik Buddy reads shell `buddyState` and `diamondEvents`.
2. Home Quest emits only `KINETIK_EVENT`.
3. Times Quest emits `practice.completed` and `mastery.unlocked`.
4. Primary Quest emits `practice.completed` and `mastery.unlocked`.
5. Code Quest Kids and Clock Quest migrate from `EARN_GEMS`.

Why:

```text
household contribution + kids learning + Buddy reaction
```

This proves the Family Companion OS.

### Wave 2 - Make family operations valuable

Wire:

- Grocery Run
- Kitchen Buddy
- Calendar actions
- Loop Coach
- Decision Coach

Why:

```text
care + routines + reflection
```

This makes Kin feel useful to parents.

### Wave 3 - Add play and sport

Wire:

- Padel Matchday
- Padel Academy
- Basketball Coach
- Tennis Coach
- Guitar Coach
- Circle Chess
- Code Clash
- Emoji Party
- Ladder & Snake / Choice Trail
- Fortune Wheel / Circle Spinner

Why:

```text
Move Energy + Play Energy + leaderboards
```

### Wave 4 - Story and sidecars

Wire carefully:

- Moments
- Cinema
- World Cup Arena
- Strata

Why:

```text
Story Energy + seasonal badges
```

Do not let sidecars inflate the main wallet.

## 10. App Event Payloads

### Ladder & Snake / Choice Trail Round

```js
{
  type: "KINETIK_EVENT",
  appId: "wisdom-trail",
  payload: {
    eventType: "game.round.completed",
    personId: "person_123",
    sourceRecordId: "ladder_round_2026-06-18_01",
    amount: 2,
    metadata: {
      title: "Ladder & Snake round",
      boardTheme: "choices",
      result: "finished",
      reflectionSaved: false
    }
  }
}
```

If the player writes or chooses a meaningful reflection, the app can emit
`reflection.saved` instead of, or in addition to, the tiny round event.

### Fortune Wheel Action

```js
{
  type: "KINETIK_EVENT",
  appId: "lucky-spin",
  payload: {
```

## 11. Data Schema

### Prototype Local Shape

Currently local:

```text
DB.d.diamondEvents[]
DB.d.agentActions[]
DB.d.buddyState[circleId]
```

This is acceptable for the POC but should become persisted.

### Google Sheets / Apps Script Baseline

Implemented in `Code.gs`:

```text
KinetikEvents
DiamondEvents
BuddyState
```

Planned later:

```text
BuddyQuests
BuddyInventory
LeaderboardSnapshots
EconomyPolicies
```

The Apps Script action `recordKinetikEvent` is the current server-side referee.
It validates circle membership, accepts only known event types, dedupes repeated
`sourceRecordId` rewards, applies daily caps, writes `DiamondEvents`, and updates
`BuddyState` by `circleId` and `personId`.

### Supabase Target

```sql
kinetik_events (
  id uuid primary key,
  circle_id uuid not null,
  person_id uuid,
  app_id text not null,
  event_type text not null,
  source_record_id text,
  proposed_amount int default 0,
  metadata jsonb default '{}',
  status text default 'accepted',
  created_at timestamptz default now()
);

diamond_events (
  id uuid primary key,
  kinetik_event_id uuid references kinetik_events(id),
  circle_id uuid not null,
  person_id uuid,
  app_id text not null,
  event_type text not null,
  energy text not null,
  amount int not null,
  cap_applied boolean default false,
  reason text,
  created_at timestamptz default now()
);

buddy_state (
  circle_id uuid primary key,
  diamonds int default 0,
  xp int default 0,
  level int default 1,
  mood text default 'curious',
  energy jsonb default '{}',
  by_person jsonb default '{}',
  last_reaction text,
  updated_at timestamptz default now()
);

economy_policies (
  id uuid primary key,
  app_id text,
  event_type text not null,
  energy text not null,
  default_amount int not null,
  max_per_event int not null,
  daily_cap int not null,
  approval_required boolean default false,
  active boolean default true
);
```

## 13. Buddy Shop And Sinks

The economy needs sinks so diamonds matter.

Spendable:

- avatar accessories
- room themes
- background worlds
- celebration effects
- family badges
- Cinema frames
- story templates
- quest boosts that do not affect real rewards

Do not sell:

- approval bypass
- uncapped rewards
- fake mastery
- parent permissions

## 14. Required Build Changes

### Shell

- Make `KinEngine.rules` use this economy policy table.
- Accept `KINETIK_EVENT` only for approved event types.
- Convert old `EARN_GEMS` bridge messages into warnings or migration shims.
- Persist `KinetikEvents`, `DiamondEvents`, and `BuddyState` through Apps Script.
- Add a Buddy economy panel in the shell or Buddy app.

### Apps

Each app must declare:

- event types it emits
- source record IDs
- person ID
- amount proposal
- metadata title and skill/category

Each app must stop treating local gems as the authority.

### Buddy

Buddy must read:

- total diamonds
- level
- mood
- energy map
- latest diamond events
- by-person contribution
- quests
- inventory

## 15. Current State Snapshot

Current shell:

- `KinEngine` exists.
- Apps emit `KINETIK_EVENT`; the shell now tries Apps Script first through
  `recordKinetikEvent`, then falls back to the local POC if the deployed backend
  is old or offline.
- `KinetikEvents`, `DiamondEvents`, and `BuddyState` are defined in `Code.gs`.
- `diamondEvents`, `agentActions`, and `buddyState` still exist locally as cache
  and offline fallback.
- `KinSpine` can already emit `calendar.created`.
- Home Quest partially emits legacy diamond/chore events.
- Many apps still use old `EARN_GEMS` or local gems.

This means the economy is now on the Apps Script path without redesigning the
whole app. The main work left is standardizing app emissions, migrating legacy
local gems, and adding server-side spend/inventory sinks.

## 16. Decision

The first real economy release should be:

```text
Home Quest + Times Quest + Primary Quest + Kinetik Buddy
```

This creates the simplest complete loop:

```text
help at home
  -> learn something
  -> Buddy grows
  -> parent sees progress
  -> Kin suggests the next useful action
```
